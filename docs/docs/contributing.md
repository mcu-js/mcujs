# Contributing

Contributions are welcome. Open an issue for proposed changes and follow the existing coding style in `src/`.

## Good first steps

- Improve docs or examples
- Add a small test to the Bun harness
- Expand an existing hardware API

## Development flow

1. Build a [UF2](./glossary.md#uf2) for your board.
2. Flash and test on hardware.
3. Run the end-to-end test harness when possible.
4. Share logs or notes in your PR if hardware is involved.

## Key terms

- [Firmware](./glossary.md#firmware)
- [Runtime](./glossary.md#runtime)
- [UF2](./glossary.md#uf2)
